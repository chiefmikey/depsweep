## Learnings

1. **ESM projects need `--input-type=module`** for inline `node -e` scripts with `import` syntax. This is cleaner than `--experimental-vm-modules`.
2. **`actions/checkout@v5` doesn't exist** — someone (likely AI) generated it speculatively. Always verify action versions.
3. **`npm audit --json` writes to stdout, not a file** — `npm audit --json > file.json` works but `npm audit` exits non-zero on vulnerabilities, so the redirect to the json file needs `|| true` to succeed.
4. **CLI tools legitimately use console.log** — a blanket "no console.log in src/" check is wrong for CLI projects.
5. **Coverage threshold checks were running `npm run test:coverage` a second time** — reading `coverage-summary.json` is much faster and uses data already generated.
6. **Snyk requires a token** — conditional on `env.SNYK_TOKEN != ''` is the right pattern.
